## 📐 Архитектурные заметки

### Как компоненты взаимодействуют

```
┌─────────────────────────────────────────────────┐
│                   MainWindow                     │
│  (UI-события → ViewModel команды)               │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌──────────────┐    ┌──────────────────────┐   │
│  │ MainViewModel │───→│ TabViewModel (×N)    │   │
│  │ (оркестратор) │    │ (WebView2 обёртка)   │   │
│  └──────┬───────┘    └──────────┬───────────┘   │
│         │                       │                │
│    ┌────┴────┐            ┌─────┴──────┐        │
│    │Services │            │ AdBlock    │        │
│    │         │            │ (фильтрация│        │
│    │-Stealth │            │  запросов) │        │
│    │-History │            └────────────┘        │
│    │-Bookmark│                                  │
│    │-Search  │            ┌──────────────┐      │
│    │-Settings│            │DownloadService│     │
│    │-AdBlock │            │(загрузки)    │      │
│    └─────────┘            └──────────────┘      │
│                                                 │
└─────────────────────────────────────────────────┘
```

### Ключевые связи:
- **MainViewModel → TabViewModel**: создаёт, передаёт AdBlockService
- **TabViewModel → WebView2**: подписка на WebResourceRequested (AdBlock)
- **SettingsService ↔ Все сервисы**: настройки загружаются при старте
- **SettingsPage → MainViewModel**: DataContext = ViewModel, доступ ко всем сервисам

### Где добавлять новый код:
- **Новый сервис** → `Services/` + свойство в MainViewModel
- **Новая модель** → `Models/`
- **Новый UI** → `Views/` или XAML секция в SettingsPage
- **Новая команда** → RelayCommand/AsyncRelayCommand в ViewModel
