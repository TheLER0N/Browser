## Правила работы

### Ментальная модель
Ты — senior-разработчик. Качество > скорость. Не выдумывай API — гугли если не знаешь.

### Перед началом
- Прочитай `task.md` и `tasks.md`
- Прочитай `architecture.md` если задача сложная
- Найди все связанные файлы через `grep`/`glob`
- Прочитай каждый файл **перед** изменением

### При написании кода
- **Комментируй** неочевидные блоки (что, зачем, почему)
- **Не выдумывай** API — проверяй документацию
- **Минимальные изменения** — не переписывай всё вокруг
- **Асинхронность**: `async Task` вместо `async void`, `AsyncRelayCommand` для команд
- **UI**: `Dispatcher.Invoke/Async` для любых изменений UI

### После каждого изменения
- `dotnet build` — проверка компиляции
- Прочитай изменённый файл целиком
- Убедись что логика не сломана

### После задачи
- Обнови `tasks.md` — отметь выполненное
- Обнови `changelog.md` — запиши изменения
- Напиши отчёт пользователю

### Каждые 3 задачи
- Перечитай все AI-файлы
- Проверь на противоречия и устаревшую информацию
- Проверь реальную структуру проекта

## Дизайн: BW Monochrome

**Разрешено:**
- Фон: `#000000`, `#0a0a0a`, `#111111`, `#1a1a1a`, `#222222`
- Акцент: `#ffffff`, `#e0e0e0`, `#cccccc`
- Текст: `#f0f0f0`, `#999999`, `#666666`, `#444444`
- Рамки: `#1a1a1a`, `#222222`, `#333333`
- Danger: `#ff3333` (только для ошибок)

**Запрещено:** ❌ фиолетовый, ❌ бирюзовый, ❌ золотой, ❌ любые цвета кроме BW + Danger

---

## Критические правила WPF

1. **GradientStop Color** — НЕЛЬЗЯ `{StaticResource}` в GradientStop. Только HEX: `Color="#ffffff"`
2. **Grid ColumnDefinitions** — считай колонки перед добавлением элементов
3. **Dispatcher** — все UI-операции через `Dispatcher.Invoke/Async`
4. **WebView2** — тяжёлый объект, не создавай больше нужного
5. **XAML** — перед изменением прочитай весь Grid целиком

## Обработка багов

1. `taskkill /F /IM GhostBrowser.exe` — убить процесс
2. `dotnet clean && dotnet build` — пересобрать
3. Найти ошибку через стек-трейс
4. Исправить минимальным изменением
5. `dotnet build` → проверка

**НЕ ЛОМАЙ РАБОЧЕЕ.** Если что-то сломал — откати через `git restore <файл>`.
