## 📂 Система файлов для выдачи задач

```
AI/
├── task.md           ← ПОЛЬЗОВАТЕЛЬ: сюда пишешь задачу простыми словами
├── tasks.md          ← ИИ: здесь детализированные подпланы с файлами и статусами
├── ideas.md          ← БАНК ИДЕЙ: все возможные фичи с ID (STEALTH-001, BYPASS-001 и т.д.)
├── roadmap.md        ← ДОРОЖНАЯ КАРТА: фазы разработки
├── design.md         ← ДИЗАЙН-СИСТЕМА: цвета, шрифты, компоненты
├── security.md       ← БЕЗОПАСНОСТЬ: механизмы защиты
├── architecture.md   ← АРХИТЕКТУРА: структура проекта, компоненты
├── rules.md          ← ПРАВИЛА: как ИИ должен работать
├── onboarding.md     ← ВВОД В КУРС ДЕЛА: быстрая загрузка контекста
├── changelog.md      ← ИСТОРИЯ: что было ДО/ПОСЛЕ каждого изменения
├── bug-handling.md   ← БАГИ: как диагностировать и чинить
├── user-responses.md ← КОНТЕКСТ: ответы пользователя, состояние проекта
└── workflow.md       ← ЭТОТ ФАЙЛ: как пользоваться системой
```

## 🎯 Как пользователю выдавать задачи

### Способ 1: Написать задачу в task.md

Откройте `AI/task.md` и в секцию `[моя просьба]` напишите:

```
Моя задача:
Сделай STEALTH-002
```

ИЛИ

```
Моя задача:
Добавь фавиконки на вкладки
```

ИЛИ

```
Моя задача:
Почини баг: при закрытии вкладки вылетает
```

ИЛИ

```
Моя задача:
Предложи что делать дальше
```

### Способ 2: Написать идеи из ideas.md

| ID | Идея | Описание |
|----|------|----------|
| `STEALTH-002` | Паник-кнопка | F12 → сворачивает всё, открывает Google, стирает историю |
| `STEALTH-003` | Туннельный режим | Встроенный VPN/SOCKS5 туннель |
| `BYPASS-001` | Авто-обход | Браузер САМ определяет блокировку и пробует DoH → прокси → мост |
| `PRIVACY-001` | Автоочистка | Автоудаление истории/cookies/кэша по таймеру |
| `PRIVACY-002` | Анти-трекинг | Блокировка Google Analytics, Facebook Pixel, fingerprint |
| `DESIGN-001` | Адаптивные темы | Браузер подстраивает цвета под сайт |
| `DESIGN-002` | Вертикальные вкладки | Вкладки слева вертикально |
| `PERF-001` | Менеджер памяти | "Засыпание" неактивных вкладок |

### Способ 3: Попросить предложить

Напишите в task.md:
```
Предложи что делать дальше
```

ИИ покажет ТОП идей из `ideas.md` с приоритизацией.

## 🔄 Как ИИ обрабатывает задачу

```
1. Пользователь пишет в task.md → "Сделай STEALTH-002"
   ↓
2. ИИ читает ideas.md → находит STEALTH-002 → понимает что нужно
   ↓
3. ИИ читает все AI-файлы → изучает контекст
   ↓
4. ИИ создаёт подпланы в tasks.md → каждый подплан = маленький шаг
   ↓
5. ИИ показывает план пользователю → ждёт подтверждения
   ↓
6. ИИ выполняет подпланы ПО ОДНОМУ → обновляет статус в tasks.md
   ↓
7. После каждого подплана → dotnet build → проверка
   ↓
8. Когда всё готово → отчёт пользователю
   ↓
9. Обновляет changelog.md, tasks.md, task.md
```

## 📋 Формат подплана в tasks.md

```
### Подплан X.Y: [Название]
- **Файлы:** [список файлов которые будут изменены]
- **Что сделать:** [конкретное описание, 1-2 предложения]
- **Как проверить:** [команда или действие для проверки]
- **Статус:** ⏳ pending / 🔄 in_progress / ✅ completed / ⚠️ blocked / ❌ failed
```

## 🚨 Правила выполнения

### Подплан считается МАЛЕНЬКИМ если:
- ✅ Затрагивает 1-3 файла (не больше)
- ✅ Изменяет <50 строк кода
- ✅ Имеет конкретный результат
- ✅ Можно проверить отдельно

### ИИ ОБЯЗАН:
- ✅ Прочитать ВСЕ AI-файлы перед началом
- ✅ Найти все связанные файлы кода (grep/glob)
- ✅ Прочитать каждый файл перед изменением
- ✅ Записать подпланы в tasks.md
- ✅ Помечать статус каждого подплана
- ✅ Собирать проект после каждого изменения (`dotnet build`)
- ✅ Проверять себя после каждого подплана
- ✅ Обновить AI-файлы после завершения
- ✅ Написать отчёт пользователю

### ИИ НЕЛЬЗЯ:
- ❌ Выполнять большую задачу целиком без разбивки
- ❌ Пропускать запись подпланов в tasks.md
- ❌ Менять >3 файлов в одном подплане
- ❌ Менять >50 строк в одном подплане
- ❌ Пропускать проверку после подплана
- ❌ Начинать следующий подплан, не завершив текущий
- ❌ Использовать цвета кроме BW палитры (кроме Danger #ff3333)

## 📊 Статусы подпланов

| Символ | Статус | Описание |
|--------|--------|----------|
| ⏳ | pending | Ещё не начат |
| 🔄 | in_progress | Выполняется сейчас |
| ✅ | completed | Выполнен и проверен |
| ⚠️ | blocked | Заблокирован (указать причину) |
| ❌ | failed | Провален (указать ошибку) |

## 🔁 Периодическая проверка AI-файлов

**После каждых 3 выполненных задач ИИ обязан:**
1. Перечитать ВСЕ 13 файлов из папки `AI/`
2. Проверить на противоречия между файлами
3. Проверить на устаревшую информацию
4. Проверить реальную структуру проекта — совпадает ли с описанной
5. Исправить все найденные несоответствия
6. Собрать проект — `dotnet build`
7. Вывести отчёт о диагностике

## 💡 Советы пользователю

### Как эффективно давать задачи:

1. **Будьте конкретны** — "Добавь фавиконки на вкладки" лучше чем "Улучши вкладки"
2. **Используйте ID из ideas.md** — "Сделай STEALTH-002" → ИИ сразу поймёт
3. **Опишите баг подробно** — "При закрытии вкладки вылетает" + шаги воспроизведения
4. **Просите предложения** — "Предложи что делать дальше" → ИИ покажет приоритеты

### Чего избегать:

1. ❌ Не пишите vague задачи — "Сделай лучше" → непонятно что именно
2. ❌ Не давайте несколько задач в одной строке — разбейте на отдельные записи
3. ❌ Не пропускайте проверку после каждого изменения — тестируйте

## 🎨 Дизайн-правила (BW Monochrome)
