# 🤖 Автопротокол работы AI с проектом KingBrowser

> При получении задачи AI **АВТОМАТИЧЕСКИ** выполняет этот протокол.  
> Пользователь только пишет задачу — всё остальное AI делает сам.

---

## АВТОМАТИЧЕСКИЙ ПРОТОКОЛ (выполняется при КАЖДОЙ задаче)

### Шаг 1 — Контекст (молча, без вопросов)
```
1. ПРОЧЕСТЬ WORKFLOW.md → вспомнить правила работы
2. ПРОЧЕСТЬ RULES.md → вспомнить конвенции и стандарты
3. ПРОЧЕСТЬ WALKTHROUGH.md → понять какие файлы затрагивает задача
4. ПРОЧЕСТЬ KNOWN_ISSUES.md → проверить связанные проблемы
5. ПРОЧЕСТЬ SPRINT.md → проверить есть ли уже такая задача в работе
```

### Шаг 2 — Анализ и разбиение
```
1. Определить затронутые файлы
2. Оценить сложность:
   - Маленькая (1-2 файла, 1-3 шага) → выполнять сразу
   - Большая (>3 файлов, >5 шагов, новые компоненты) → разбить на подзадачи
3. Составить план выполнения
4. Показать план пользователю (кратко)
```

### Шаг 3 — Выполнение
```
1. Выполнять по одной подзадаче
2. После каждой — проверка компиляции
3. НЕ менять >5 файлов за раз
4. НЕ писать >200 строк без проверки
```

### Шаг 4 — Обновление AI-файлов (АВТОМАТИЧЕСКИ)
```
ЗАДАЧА ДОБАВЛЕНА:
  → TASKS.md: добавить задачу в список
  → SPRINT.md: добавить в активные если делаем сейчас

ЗАДАЧА В РАБОТЕ:
  → SPRINT.md: обновить прогресс
  → TASKS.md: обновить статус

ЗАДАЧА ЗАВЕРШЕНА:
  → TASKS.md: отметить ✅
  → SPRINT.md: обновить прогресс-бар
  → KNOWN_ISSUES.md: удалить если проблема решена

НОВАЯ ПРОБЛЕМА ОБНАРУЖЕНА:
  → KNOWN_ISSUES.md: добавить проблему

ВАЖНОЕ РЕШЕНИЕ ПРИНЯТО:
  → NOTES.md: записать решение
```

### Шаг 5 — Отчёт
```
Формат ответа:
**Задача:** [название]
**Подзадачи:** [список если большая]

### Сделано:
1. [действие] → [файл]
2. [действие] → [файл]

### AI-файлы обновлены:
- [какие файлы обновлены и что changed]

### Проверить:
- [ ] Компиляция: dotnet build
- [ ] [функциональность]

### Замечания:
[если есть]
```

| Технический долг выявлен | TASKS.md | Добавить в техдолги |

## ЗАПРЕТЫ (АБСОЛЮТНЫЕ)

```
❌ НЕ спрашивать «какой файл посмотреть?» — смотри WALKTHROUGH.md
❌ НЕ спрашивать «как разбить задачу?» — разбивай сам по правилу 2.2
❌ НЕ писать код без чтения существующего
❌ НЕ менять >5 файлов за раз
❌ НЕ удалять код без подтверждения
❌ НЕ генерировать код при незнании API — сначала web_search
❌ НЕ трогать App.xaml без чёткой необходимости
❌ НЕ пропускать обновление AI-файлов после задачи
```

## МИНИМАЛЬНЫЙ ДИАЛОГ

```
Пользователь: «Добавь тёмную тему»

AI (правильно):
→ Молча читает WORKFLOW.md, RULES.md, WALKTHROUGH.md
→ Определяет: Models/AdvancedSettings.cs, Views/SettingsPage.xaml, App.xaml
→ Разбивает: модель → настройки → UI → стили
→ Показывает краткий план
→ Выполняет
→ Обновляет TASKS.md, SPRINT.md
→ Отчитывается

AI (неправильно):
→ «Какой файл мне посмотреть?»
→ «Как вы хотите разбить задачу?»
→ «Где искать стили?»
```
