# KingBrowser — Инструкции для AI

> Этот файл загружается автоматически при каждом обращении к проекту.

## 🤖 При получении задачи — ОБЯЗАТЕЛЬНЫЙ ПРОТОКОЛ:

1. **Прочитать** `AI/AUTO_WORKFLOW.md` — это главный автосценарий, выполнять его строго
2. Следовать правилам из `AI/WORKFLOW.md`
3. Использовать `AI/RULES.md` для конвенций кода
4. Использовать `AI/WALKTHROUGH.md` для навигации по файлам

## 📁 Папка AI (все файлы важны):

| Файл | Когда использовать |
|------|-------------------|
| `AI/AUTO_WORKFLOW.md` | **ЧИТАТЬ ПЕРВЫМ** — автосценарий работы |
| `AI/WORKFLOW.md` | Правила работы с кодом |
| `AI/RULES.md` | Конвенции, MVVM, стандарты C#/WPF |

| `AI/PLAN.md` | План развития проекта |

## 📌 О проекте

- **C# 12, .NET 7 WPF, MVVM**
- **WebView2** — Chromium рендеринг
- **Stealth mode** — скрытие окна от OBS/Discord/Zoom
- **Инкогнито, загрузки, DNS, закладки, история**

## ⚡ Главное правило

**Пользователь пишет задачу → AI делает всё сам:**
- Читает нужные AI-файлы
- Разбивает задачу
- Выполняет по шагам
- Обновляет AI-файлы
- Отчитывается кратко

**НЕ спрашивать:** «какой файл посмотреть?», «как разбить?», «что делать?»
**ДЕЛАТЬ:** молча читать AUTO_WORKFLOW.md и действовать по протоколу
